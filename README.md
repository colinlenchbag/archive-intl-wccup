# archive-intl-wccup

该仓库用于归档和托管若干独立 HTML 页面。这些页面不针对任何单个域名或具体网站，仅为保存和展示特定时间点的网页内容而设立。

## 项目简介

本仓库的主要目的是对多个来源的 HTML 页面进行集中存档，并提供基本的访问路径。所有页面均为静态 HTML 文件，不包含后端逻辑或数据库依赖。

仓库本身不提供任何内容推荐、评价或导向，仅作为资料存储与展示用途。

## 目录说明

```
/
├── pages/          # 存放归档的 HTML 页面文件
├── assets/         # 页面引用的静态资源（如 CSS、JS、图片等）
├── index.html      # 仓库根目录下的页面索引（可选）
└── README.md       # 本文件
```

- `pages/` 目录下每个文件夹代表一个独立归档单元，文件夹名称对应归档主题或代号。
- `assets/` 目录用于存放多个页面可能共用的资源，也可在每个归档单元内独立存放资源。
- 索引页面 `index.html` 用于列出所有归档页面链接，方便导航。

## 页面归档说明

- 归档的 HTML 页面均来自公开可访问的网页，在归档时已尽力保留原始结构与样式。
- 归档内容可能包含外部资源引用（如字体、脚本、图片等），这些资源可能因原服务器变动而失效。
- 部分页面可能经过微调（如修复相对路径、移除跟踪代码等），以保持基本显示与可读性。
- 归档时间戳及来源信息通常记录在页面注释或文件夹说明文件中。

## 维护说明

- 本仓库不定期更新，新增归档页面会以独立文件夹形式添加。
- 不保证所有页面长期可用或样式完全一致，外部资源失效属正常现象。
- 欢迎提交 Issue 或 Pull Request 来报告链接失效、内容错误或建议新增归档。
- 请注意，归档内容不代表仓库维护者观点，仅为资料保存。

## 使用方式

- 直接通过 GitHub Pages 或本地静态服务器浏览 `index.html` 或 `pages/` 下的页面。
- 也可直接访问 `pages/<归档目录>/index.html` 查看单个归档页面。
